# Synthetic document generator

Закрывает требования: генерирует тестовые документы шести типов в форматах PDF / DOCX / XLSX / CSV, плюс корзину «негативных» документов для проверки фильтров. Поддерживает multi-counterparty (до 4 сторон в одном документе) и пять edge-кейсов для подготовки тест-корпуса под multi-entity NER.

## Установка

```bash
cd tools/generator
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

## Запуск

```bash
# 5 документов всех MVP-типов во всех форматах
python -m generator --out ./out --seed 42

# Только счета-фактуры в PDF, 20 шт., 3 контрагента, детерминировано
python -m generator --type invoice --format pdf --count 20 --parties 3 --seed 1 --out ./out

# Multi-counterparty стресс-тест: 4 стороны + все edge-кейсы
python -m generator \
  --type invoice --type act --type contract \
  --type waybill --type upd --type payment_order \
  --parties 4 \
  --edge-case ip_forms --ip-mode guaranteed \
  --edge-case missing_optional --missing-mode guaranteed \
  --edge-case multiline_names --edge-case reordered_blocks \
  --count 1 --out ./out --seed 7

# Все 6 типов + 10 негативных документов
python -m generator \
  --type invoice --type act --type contract \
  --type waybill --type upd --type payment_order \
  --include-junk 10 --count 3 --out ./out --seed 7
```

## Что генерируется

- **invoice** (счёт-фактура): pdf, docx, xlsx, csv — с таблицей позиций, НДС 22%
- **act** (акт): pdf, docx — без позиций, общая сумма
- **contract** (договор): pdf, docx — стороны, предмет, срок
- **waybill** (ТОРГ-12): pdf, xlsx, csv — таблица позиций
- **upd** (УПД): pdf, xlsx — формат счёта
- **payment_order** (платёжное поручение): pdf, docx — плательщик, получатель, назначение

Все ИНН валидны по контрольной сумме ФНС: 10 цифр для юрлиц, 12 — для ИП. Контрагенты — случайные комбинации формы (ООО/ОАО/ЗАО/ПАО/АО) и имени из встроенного словаря. Даты — случайные в 2026 году. Суммы — целые рубли + копейки.

## Multi-counterparty (`--parties N`)

По умолчанию `N=2` (минимум для большинства документов). Hard cap = 4 сверху. Каждый тип имеет собственный потолок — при `N` выше потолка генератор молча клипает.

| Тип             | Cap | Роли (по убыванию важности)                                                |
|-----------------|-----|----------------------------------------------------------------------------|
| `invoice`       | 4   | Продавец, Покупатель, Грузоотправитель, Грузополучатель                     |
| `upd`           | 4   | Продавец, Покупатель, Грузоотправитель, Грузополучатель                     |
| `waybill`       | 4   | Грузоотправитель, Грузополучатель, Поставщик, Плательщик                    |
| `contract`      | 4   | Сторона 1, Сторона 2, Поручитель, Бенефициар                                |
| `act`           | 3   | Исполнитель, Заказчик, Субподрядчик                                         |
| `payment_order` | 2   | Плательщик, Получатель                                                      |

ИНН среди сторон одного документа гарантированно уникальны. 5+ ролей в реальных бизнес-документах не встречаются — поэтому жёсткий потолок 4.

## Edge-кейсы (`--edge-case ...`, повторяемый)

Каждый кейс независимо инжектируется при сборке `parties` и не задан по умолчанию. Каждый бьёт по конкретной слабости regex-NER, которую LLM-NER (Phase 3) должен будет решить.

| ID                  | Что делает                                                                  | Какой баг проявляет                                                 |
|---------------------|-----------------------------------------------------------------------------|---------------------------------------------------------------------|
| `ip_forms`          | стороны становятся ИП. Интенсивность задаётся `--ip-mode` (см. ниже)         | regex присваивает ИП чужой КПП от соседнего юр.лица                  |
| `missing_optional`  | у юр.лиц обнуляется КПП и/или город. Интенсивность задаётся `--missing-mode` | устойчивость pipeline к разреженным шаблонам                        |
| `multiline_names`   | у ~45% юр.лиц длинное дефисное имя разрывается переносом строки             | `COUNTERPARTY_RE` имеет `\n` в исключении и обрежет имя на первой строке |
| `reordered_blocks`  | стороны рендерятся в обратном порядке (Покупатель → Продавец)               | regex берёт `inns[0]` и присваивает роли по позиции — зеркальное role assignment |

### Режимы ИП (`--ip-mode`, действует только при `--edge-case ip_forms`)

| Режим         | Поведение                                                              |
|---------------|------------------------------------------------------------------------|
| `random` (по умолчанию) | каждая сторона с вероятностью ~35% становится ИП          |
| `guaranteed`  | минимум одна сторона — ИП, остальные — random ~35%                     |
| `all`         | все стороны документа — ИП                                             |

### Режимы пропусков полей (`--missing-mode`, действует только при `--edge-case missing_optional`)

| Режим         | Поведение                                                              |
|---------------|------------------------------------------------------------------------|
| `random` (по умолчанию) | у каждого юр.лица ~20% вероятность без КПП, ~15% без города |
| `guaranteed`  | минимум одно юр.лицо принудительно без КПП и без города; остальные — random |
| `all`         | все юр.лица без КПП и без города (стресс-тест разреженных бланков)      |

## Негативные документы (F-G5)

`--include-junk N` создаёт N файлов в `./out/junk/`:

- **junk-text** (.txt): нерелевантный текст (лекция, открытка, записка) — должен попасть в `validated_with_errors` с кодом `unknown_document_type` после реализации F-T13.
- **junk-empty** (.txt): пустой файл — должен отвергнуться extract'ом с 400.
- **junk-garbled** (.pdf): «битый» PDF — должен либо отвергнуться парсером, либо дать пустой текст и попасть в F-T13.
- **junk-wrong-ext** (.docx): plain-text с расширением .docx — провоцирует ошибку парсера.

> На Linux для PDF может потребоваться `apt install fonts-dejavu-core`. На macOS работает из коробки.

## Загрузка в пайплайн

```bash
TOKEN=$(curl -s -X POST http://localhost:8000/api/auth/login \
  -d 'username=admin&password=admin' | jq -r .access_token)

for f in ./out/invoice/*.pdf; do
  curl -s -X POST http://localhost:8000/api/documents \
    -H "Authorization: Bearer $TOKEN" \
    -F "file=@$f"
  echo
done
```

Через UI: Админка → «Сгенерировать» (slider контрагентов + чекбоксы edge-кейсов + кнопка «Multi smoke-test» для быстрого стресс-теста).
